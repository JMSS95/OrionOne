# Requisitos do Sistema OrionOne

## Requisitos Funcionais

### RF01 - Gestão de Tickets

O sistema deve permitir criar, visualizar, editar e eliminar tickets de suporte.

### RF02 - Autenticação

O sistema deve suportar autenticação segura com diferentes níveis de acesso.

### RF03 - Sistema de Comentários

Utilizadores devem poder adicionar comentários públicos e internos aos tickets.

## Requisitos Não Funcionais

### RNF01 - Performance

O sistema deve responder em menos de 2 segundos em 95% dos casos.

### RNF02 - Segurança

Todas as passwords devem ser armazenadas com hash bcrypt.

### RNF03 - Usabilidade

Interface deve ser responsiva e funcionar em dispositivos móveis.
